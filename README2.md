## описание реализации
**структуры данных**
Для графа в коде используются три представления:
Матрица смежности — двумерный массив размером V×V, где элемент [i][j] = 1, если между вершинами есть ребро. Удобна для быстрой проверки смежности за O(1), но занимает много памяти.
Матрица инцидентности — массив размером V×E, связывающий вершины с рёбрами. Применяется для задач, где важны сами рёбра.
Список смежности — словарь {вершина: [соседи]}, используется внутри поиска компонент связности. Наиболее экономный вариант для разреженных графов.
Для дерева реализованы:
Node — базовый класс узла с полями left, right, val.
BST (бинарное дерево поиска) — структура, где левое поддерево содержит меньшие значения, правое — большие. Представлена классом BST с корнем root.
Куча (Heap) — полное бинарное дерево, представленное массивом, используется в сортировке heap_sort.

**Сложность алгоритмов (Big O)**
***Графовые алгоритмы:***
get_adjacency_matrix — O(V²) по времени и памяти (создание квадратной матрицы).
get_incidence_matrix — O(V×E) (матрица вершины×рёбра).
find_connected_components — O(V + E), так как DFS посещает каждую вершину и ребро один раз.
***Алгоритмы BST:***
insert, search, delete — O(log n) в среднем (сбалансированное дерево), O(n) в худшем случае (вырожденное дерево в список).
inorder — O(n), обходит все узлы ровно один раз.
***Сортировка:***
heapify — O(log n) (спуск по высоте кучи).
heap_sort — O(n log n) всегда, дополнительная память O(1) (сортировка на месте).
